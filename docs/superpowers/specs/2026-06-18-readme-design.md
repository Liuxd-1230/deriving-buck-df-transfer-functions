# README Design

## Audience and goal

Serve engineers who discover the repository on GitHub and need to decide quickly whether the skill fits their Buck control problem, install it without Zotero, run one known-model example, and understand what evidence does or does not prove.

## Structure

Lead with the v0.3 outcome and the event-first safety rule. Follow with a compact decision flow, supported-model table, installation, three workflows, validation evidence, repository map, and explicit non-goals. Use Chinese prose with original English identifiers and executable commands.

## Claim discipline

Keep registered paper models separate from protocol-derived candidates. State that protocol-checker success proves completeness rather than physical correctness, and that no switching simulation or independent agent forward-test validates a new v0.3 model yet.
