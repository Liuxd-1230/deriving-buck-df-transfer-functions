# Buck 描述函数推导与验证流程

## 1. 先判断模型是否适用

记录以下事实后再写公式：

- Buck、CCM、单相或多相、同步或非同步
- COT current-mode、RBCOT、V²、固定频率或其他控制律
- 固定导通时间 `T_on`、稳态周期 `T_s`、`D=T_on/T_s`
- 比较器输入、外部 ramp、内部 RC ramp、采样和传播延时
- 目标为 `Gvc`、`Gvg`、`Zout`、内环 return ratio 或完整 `Tloop`
- 有效频率上限和忽略的寄生参数

DCM、脉冲跳跃、限流、饱和、强非线性大扰动不属于本流程。多相脉冲发生重叠时，不得把单相 DF 系数逐相相加。

## 2. 固定符号和扰动方向

使用 `x(t)=X+Re{x_hat exp(st)}`，其中频率响应时 `s=jω`。约定：

- `v_g`：输入电压扰动
- `u_c`：控制器送入调制器的控制扰动
- `i_load`：从输出节点流出的负载扰动
- `Zout=-v_o_hat/i_load_hat`，因此正值代表负载增加导致输出下降
- 所有反馈符号直接写进 DF 系数，不在最终式中凭直觉补负号

## 3. 写稳态轨迹和开关边沿条件

先得到一个完整稳态周期内的 `i_L(t)`、`v_o(t)`、ramp 与比较器输入。用事件方程

`F(x(t_k), r(t_k), u_c(t_k), t_k)=0`

定义每个可移动边沿。边沿的一阶时间扰动满足事件敏感度关系

`delta_t_k = -delta_F(t_k) / F_dot_0(t_k)`。

随后把边沿时间变化转换为开关函数基波、脉冲面积或等效 duty/周期扰动。高频 DF 中的 `d_hat` 应理解为送入功率级的等效归一化开关函数扰动，不一定等于低频平均模型里的瞬时 duty。COT 的 `T_on` 固定并不代表 `d_hat=0`；变化通常进入关断时间、开关周期和频率。

若精确 DF 出现 `exp(-s T_on)`、`exp(-s T_s)`、谐波求和或无限阶项，先保留精确形式。只有在声明频率范围后才能做 Padé/Taylor 或低阶多项式近似。

## 4. 提取通用调制关系

将边沿推导整理为

`d_hat = a_c(s)u_c_hat + a_g(s)v_g_hat + a_o(s)v_o_hat + a_i(s)i_L_hat`。

其中 `a_*` 必须来自前一步。外部 ramp 会改变边沿总斜率并可能带来与 `T_s` 有关的动态；内部 RC ramp/DC extractor 要保留其滤波传函；RBCOT 的输出纹波路径不能和功率级 ESR 路径重复计算。

## 5. 与 Buck 功率级联立

使用 [formula-patterns.md](formula-patterns.md) 的矩阵方程。先保留符号解，再代入元件值。运行：

```bash
python scripts/df_buck_sympy.py derive --case case.json --out derivation.md
python scripts/df_buck_sympy.py check --case case.json
```

脚本负责线性代数、化简、DC 极限和可选数值极点；它不能判断 `a_*` 是否由正确开关事件得到。

## 6. 分层验证

### A. 符号与结构

- 检查单位、反馈符号、分子分母阶次和传函 properness。
- 令 `s→0`，核对稳态微分增益。
- 令 `r_C→0`、`r_L→0`、ramp 或 sensing path 关闭，确认退化到对应简化模型。
- 低频 `|sT_s|<<1` 时应接近适用的平均模型；这不证明高频正确。
- 检查精确延时项是否在低阶化简前被保留。

### B. 文献基准

选择 [zotero-df-source-map.md](zotero-df-source-map.md) 中控制律最接近的论文，复现至少一个参数组。比较 DC gain、主极点/零点、ramp 变化趋势、gain peaking、交越频率和相位裕量，而非只比较最终公式外观。

### C. 开关仿真

优先使用 SIMPLIS POP/AC 或等效周期稳态小信号分析；没有 AC 注入时使用多周期正弦注入并进行基波解调。

1. 先确认仿真稳态工作点与推导完全一致。
2. 初始扰动可取控制量工作范围的约 `0.5%`，再用 `0.25%` 和 `1%` 复测；响应随幅度明显变化说明已离开小信号区。
3. 覆盖低频、预计交越附近、谐振/peaking 附近和模型声明的最高有效频率。
4. 同时比较 magnitude 与 phase，并确认注入位置没有改变原闭环工作点。

工程默认验收线：模型有效频段内 magnitude 误差不超过 `1 dB`、phase 不超过 `5°`；接近声明上限可放宽至 `3 dB/15°`。若论文或项目另有指标，以项目指标为准并记录原因。

### D. 实验或独立模型

高风险设计至少增加一种独立证据：网络分析仪扫频、硬件负载扰动辨识、sampled-data 模型或另一套开关仿真。两个实现共享同一错误假设时，不算独立验证。

## 7. 交付判定

报告必须列出：已通过的检查、未通过的检查、未执行的检查、有效频率范围、最大误差、模型失效条件。禁止只写“与仿真吻合”。
